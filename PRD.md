## 一、产品概述

### 1.1 产品定位

**AI 影视公司的全生命周期知识产权与制片管理平台。**

从剧本拆解到分镜生成、审查批注、风格定调、资产归档，覆盖 AI 影视制片的完整工作流。替代飞书+微信群+Excel+各种散落工具的碎片化管理方式。

### 1.2 目标用户

| 角色 | 典型操作 |
|------|---------|
| **制片人/管理者（你）** | 创建项目、分配任务、审查分镜、管理风格定调、查看进度和积分消耗 |
| **导演** | 审查分镜、画笔批注、确认风格方向、通过/打回镜头 |
| **美术/AI 绘师** | 查看任务、上传生成的分镜图/视频、查看批注修改意见、使用提示词库 |
| **编剧** | 上传/编辑剧本、查看分镜拆解结果 |
| **外地远程员工** | 通过微信扫码登录，远程协作 |

### 1.3 核心价值主张

1. **一个平台替代多个工具** — 不再微信群发图、飞书填表、Excel 记台账
2. **审查有痕迹** — 批注标注永久保留，改没改、谁改的一目了然
3. **风格统一有标准** — 提取风格关键词，团队对齐，不再各画各的
4. **知识产权长期沉淀** — 所有项目资产、角色、风格永久归档，跨项目复用
5. **远程协作无障碍** — 微信扫码登录，外地员工秒上手

## 二、核心功能详述（9 大场景）

### 场景 1：剧本智能拆解

**用户故事**：作为制片人，我希望上传一个剧本或分镜脚本后，系统自动拆分成一个个镜头卡片，这样我不需要手动在 Excel 里一行行填。

**功能描述**：

1. **剧本输入**
   - 支持粘贴文本、上传 .docx/.txt/.md 文件
   - 支持已有的分镜脚本格式（场次+镜头号+画面描述+台词+时长）

2. **AI 自动拆解**
   - 识别场次和镜头边界
   - 每个镜头提取：镜头编号、场景描述、台词/旁白、预估时长、机位/景别建议
   - 拆解结果以卡片形式排列在无限画布上

3. **手动调整**
   - 拖拽调整卡片顺序
   - 合并/拆分镜头
   - 手动新增空白卡片
   - 编辑任何字段

4. **无限画布**
   - 类似博思白板的自由拖拽体验
   - 支持缩放、平移
   - 卡片可自由排列或切换为列表/网格视图

**验收标准**：
- 上传一个 30 镜头的分镜脚本，AI 拆解准确率 ≥ 85%
- 拆解完成时间 < 30 秒
- 支持手动修正所有拆解结果
- 画布操作流畅（60fps 拖拽）

---

### 场景 2：镜头资产上传与浏览

**用户故事**：作为美术/AI 绘师，我需要把在即梦/可灵等平台生成的分镜图和视频上传到对应的镜头卡片里，让团队所有人都能看到。

**功能描述**：

1. **资产上传**
   - 每个镜头卡片支持上传多个版本（V1、V2、V3...）
   - 支持格式：图片（jpg/png/webp）、视频（mp4/webm）
   - 拖拽上传、批量上传
   - 自动记录：上传者、上传时间、版本号

2. **瀑布流浏览**
   - 项目所有镜头以瀑布流展示
   - 每个镜头显示最新版本的缩略图
   - 鼠标悬停：图片自动放大预览（不用点击打开）
   - 鼠标悬停视频：自动播放预览（静音，悬停即播）

3. **版本管理**
   - 点击镜头卡片展开所有版本
   - 版本对比（左右/上下分屏）
   - 标记「当前采用版本」
   - 历史版本保留不删除

4. **批量操作**
   - 批量下载选中镜头的资产
   - 批量复制资产到其他项目

**验收标准**：
- 单张图片上传 < 3 秒（10MB 以内）
- 瀑布流加载 100 张缩略图 < 2 秒
- 悬停预览响应时间 < 200ms
- 视频悬停播放无明显延迟

### 场景 3：画笔批注与审查

**用户故事**：作为导演/制片人，我需要在分镜图上直接圈出不满意的地方，写上修改意见，让美术看到后针对性修改。

**功能描述**：

1. **画笔批注工具**
   - 在图片上自由画笔标注（红色默认，可选颜色）
   - 矩形/圆形选区标注
   - 箭头指示标注
   - 文字批注（关联到标注位置）
   - 视频帧批注 — 暂停在某一帧，在该帧上标注

2. **批注管理**
   - 每条批注显示：批注者、时间、内容
   - 批注状态：未处理 → 处理中 → 已解决
   - 批注可回复（形成讨论线程）
   - 未解决批注数量角标提示

3. **镜头审查状态流转**

   ```
   待上传 → 待审查 → 需修改 → 已修改待复审 → 通过 → 已交付
   ```

   - 状态标签用不同颜色区分
   - 支持批量修改状态
   - 筛选：只看「需修改」的镜头

4. **审查标准模板（可选）**
   - 预设检查清单：构图 ☐ / 色调 ☐ / 角色一致性 ☐ / 道具准确性 ☐ / 场景匹配 ☐
   - 可自定义检查项
   - 审查时逐项勾选

5. **批注转任务**
   - 一条批注可一键转为任务
   - 指派给具体负责人
   - 在任务看板中追踪

**验收标准**：
- 画笔标注实时渲染，无延迟
- 批注保存后所有团队成员实时可见（< 1 秒同步）
- 视频帧定位精度到秒
- 批注不影响原图/原视频（叠加层）

### 场景 4：风格定调系统

**用户故事**：作为制片人，我希望在项目开始时上传几张参考图，AI 提取出风格关键词，团队所有人都按这个标准生成，保证风格统一。

**功能描述**：

1. **风格提取**
   - 上传 1-5 张参考图
   - AI 分析并提取 JSON 格式的风格参数（色调、光影、质感、氛围、艺术风格等）
   - 将 JSON 转换为人类可读的「风格关键词卡」
   - 支持中英文关键词

2. **风格指南卡**
   - 项目顶部固定展示风格指南
   - 包含：参考图缩略图 + 关键词列表 + 风格描述
   - 所有团队成员在该项目下都能看到
   - 支持手动编辑/微调关键词

3. **风格模板库（跨项目）**
   - 保存好的风格方案为模板
   - 模板命名、标签分类（赛博朋克/国风/写实/动漫...）
   - 新项目可从模板库一键应用
   - 支持手动创建模板（不一定从图片提取）

4. **风格迁移**
   - 已有风格 + 新的内容描述 → 生成融合后的提示词
   - 例如：「赛博朋克风格」+「古代宫殿场景」→ 赛博朋克宫殿提示词

**验收标准**：
- 风格提取时间 < 10 秒
- 提取的关键词覆盖：色调、光影、质感、氛围、艺术风格至少 5 个维度
- 风格模板库支持 100+ 模板
- 手动编辑后实时更新

### 场景 5：角色/道具/场景参考锁定

**用户故事**：作为导演，我需要确保同一个角色在 50 个镜头中看起来是同一个人，同一个道具在不同场景中保持一致。

**功能描述**：

1. **角色档案**
   - 每个项目可建立多个角色档案
   - 每个档案包含：角色名、角色描述、参考图（正面/侧面/全身/特写）
   - 参考图支持多张
   - 角色档案支持关联到具体镜头

2. **道具档案**
   - 同角色档案结构
   - 道具名称、描述、参考图

3. **场景档案**
   - 场景名称、描述、参考图（全景/细节）
   - 场景的光影、时间（白天/黑夜/黄昏）标注

4. **关联与对比**
   - 每个镜头卡片可关联：涉及的角色 + 道具 + 场景
   - 审查时一键调出该镜头关联的所有参考档案
   - 生成图与参考图并排对比

5. **跨项目复用**
   - 角色/道具/场景可复用到其他项目
   - 全局搜索角色库

**验收标准**：
- 每个档案支持上传 10+ 参考图
- 关联操作 < 3 次点击
- 对比视图加载 < 1 秒

### 场景 6：多项目与历史管理

**用户故事**：作为管理者，我需要管理公司所有项目（进行中和历史的），能快速找到任何一个项目的资产。

**功能描述**：

1. **项目中心（首页）**
   - 项目列表：封面图 + 项目名 + 状态 + 进度 + 最后更新时间
   - 状态分类：进行中 / 已完成 / 已归档
   - 创建新项目：项目名、描述、封面图、团队成员
   - 项目排序/筛选

2. **项目归档**
   - 完成的项目可归档
   - 归档项目只读，所有资产保留
   - 可随时取消归档重新编辑
   - 归档项目的资产可被其他项目引用

3. **全局搜索**
   - 跨所有项目搜索：按关键词、角色名、场景名、风格标签
   - 搜索结果展示来源项目
   - AI 语义搜索（"暖色调的室内场景"）

4. **项目仪表盘**
   - 镜头总数 / 已完成数 / 待审查数 / 需修改数
   - 各成员任务分配和完成情况
   - 简洁的进度条/饼图

**验收标准**：
- 支持 50+ 项目同时管理
- 搜索响应 < 2 秒
- 归档项目数据永久保留

### 场景 7：提示词资产库

**用户故事**：作为美术，我希望有一个团队共享的提示词库，好用的提示词存起来，下次直接复用，不用从头写。

**功能描述**：

1. **提示词录入**
   - 手动添加提示词（文字或 JSON 格式均可）
   - AI 从图片提取提示词（基于 JSON Studio 的功能）
   - 提示词标题、描述、标签

2. **分类与标签**
   - 预设分类：风格 / 人物 / 场景 / 道具 / 氛围 / 光影 / 特效
   - 自定义标签
   - 多标签筛选

3. **提示词卡片**
   - 展示：标题 + 预览图（如果有）+ 标签 + 关键词摘要
   - 点击展开完整提示词
   - 一键复制（方便粘贴到即梦/可灵/MJ 等平台）

4. **团队共享**
   - 所有团队成员可见
   - 可标记个人收藏
   - 使用次数统计（热门提示词排序）

5. **提示词编辑**
   - JSON 模式编辑器（语法高亮、格式化）
   - 自然语言模式编辑
   - 版本历史

**验收标准**：
- 支持 1000+ 提示词存储
- 搜索/筛选响应 < 1 秒
- 一键复制到剪贴板

### 场景 8：平台积分/额度登记台账

**用户故事**：作为管理者，我需要知道团队每个人每天在各个 AI 平台消耗了多少积分，用在哪个项目上。

**功能描述**：

1. **快捷登记**
   - 选择登记人（或默认当前用户）
   - 选择平台（即梦/可灵/MJ/Flux/ComfyUI/其他，可自定义）
   - 填写消耗数量（积分/次数/金额）
   - 选择关联项目（可选）
   - 备注（可选）
   - 操作在 10 秒内完成

2. **登记列表**
   - 按日期倒序展示
   - 筛选：按人 / 按平台 / 按项目 / 按日期范围

3. **月度报表**
   - 按人汇总：每人本月各平台消耗
   - 按项目汇总：每个项目各平台消耗
   - 按平台汇总：每个平台总消耗
   - 简洁的柱状图/饼图可视化

4. **额度预警（可选）**
   - 设置平台额度上限
   - 超过 80% 时提醒管理者

**验收标准**：
- 登记操作 < 10 秒
- 报表生成 < 3 秒
- 支持导出 Excel

### 场景 9：团队协作基础设施

**用户故事**：作为管理者，我需要外地员工也能方便登录使用，并且每个人看到自己负责的任务。

**功能描述**：

1. **微信扫码登录**
   - 首次：微信扫码注册（自动创建账号）
   - 后续：微信扫码一键登录
   - 无需记密码，无需下载 App
   - 管理员可邀请新成员（生成邀请链接/二维码）

2. **角色权限**
   - 管理员：全部权限 + 成员管理 + 项目设置
   - 导演：审查批注 + 状态变更 + 角色/道具管理
   - 美术：上传资产 + 查看批注 + 使用提示词库
   - 只读成员：只能浏览不能修改（给客户或外部协作者）

3. **任务看板**
   - 看板视图（类似 Trello）
   - 列：待分配 → 进行中 → 待审查 → 需修改 → 已完成
   - 每个任务卡片：镜头号 + 负责人 + 截止日期 + 优先级
   - 拖拽改状态
   - 按负责人筛选（"只看我的任务"）

4. **通知推送**
   - 被分配任务 → 微信通知
   - 收到新批注 → 微信通知
   - 镜头状态变更 → 微信通知
   - 可设置通知偏好（全部/仅@我/关闭）

5. **操作日志**
   - 记录所有关键操作（谁在什么时候做了什么）
   - 管理员可查看全部日志
   - 用于追溯和审计

**验收标准**：
- 微信扫码到登录成功 < 5 秒
- 看板拖拽实时同步（< 1 秒）
- 微信通知送达 < 30 秒

## 四、技术方案

### 4.1 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|---------|
| **前端框架** | Next.js 16 + React 19 | SSR 首屏快、App Router、Server Components |
| **UI 组件** | Tailwind v4 + shadcn/ui | 快速开发、一致性好、可定制 |
| **画布引擎** | React Flow 或 Konva.js | 无限画布拖拽、卡片排列 |
| **画笔批注** | Fabric.js 或 Konva.js | 自由画笔、形状、文字叠加层 |
| **视频预览** | 原生 HTML5 Video + 帧截取 | 悬停播放、帧定位批注 |
| **后端** | Cloudflare Workers + Hono | 全球边缘部署、低延迟 |
| **数据库** | Cloudflare D1 (SQLite) | 结构化数据、零运维 |
| **文件存储** | Cloudflare R2 | 图片/视频存储、无出口流量费 |
| **实时协作** | Durable Objects (WebSocket) | 批注实时同步、看板拖拽同步 |
| **AI 能力** | Gemini API (风格提取) + Workers AI (搜索/标签) | 风格分析、语义搜索 |
| **向量搜索** | Cloudflare Vectorize | 全局语义搜索 |
| **登录认证** | 微信 OAuth 2.0 | 扫码登录、无密码 |
| **通知推送** | 微信模板消息 / 企业微信 API | 任务和批注通知 |
| **部署** | Cloudflare Pages | 全球 CDN、自动 HTTPS、零运维 |

### 4.2 数据模型（核心表）

```
projects          — 项目（名称、描述、封面、状态、风格指南）
shots             — 镜头（项目ID、镜号、描述、台词、时长、状态、负责人）
assets            — 资产（镜头ID、文件URL、类型、版本号、上传者、时间）
annotations       — 批注（资产ID、画笔数据JSON、文字、批注者、状态）
annotation_replies — 批注回复
characters        — 角色档案（项目ID/全局、名称、描述）
character_refs    — 角色参考图
props             — 道具档案
scenes            — 场景档案
shot_relations    — 镜头-角色/道具/场景关联
style_templates   — 风格模板（名称、标签、关键词JSON、参考图）
prompts           — 提示词库（标题、内容、标签、使用次数）
credit_logs       — 积分登记（用户、平台、数量、项目、日期）
users             — 用户（微信OpenID、昵称、头像、角色）
tasks             — 任务（镜头ID、批注ID、负责人、状态、优先级）
activity_logs     — 操作日志
```

### 4.3 部署架构

```
用户浏览器
  ↓ HTTPS
Cloudflare Pages（前端 Next.js SSR）
  ↓ API 请求
Cloudflare Workers + Hono（后端 API）
  ├── D1（结构化数据）
  ├── R2（图片/视频文件）
  ├── Durable Objects（实时 WebSocket）
  ├── Vectorize（语义搜索）
  ├── Workers AI（标签/分类）
  └── Gemini API（风格提取）

微信 OAuth（登录）
微信模板消息（通知推送）
```

**为什么不用 Docker**：
- Cloudflare 全球 300+ 节点，外地员工访问更快
- 零运维，不用管服务器
- R2 存储无出口流量费，图片/视频量大也不贵
- 自动 HTTPS、DDoS 防护
- 比自建服务器便宜且稳定

## 七、风险与依赖

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 微信 OAuth 审核周期 | 登录功能延迟 | 先用手机号+验证码登录，OAuth 审核通过后切换 |
| D1 单库 10GB 限制 | 大项目数据量 | 按项目分库，或升级 Workers 付费计划 |
| R2 大视频上传慢 | 用户体验差 | 分片上传 + 进度条，限制单文件 500MB |
| AI 拆解准确率不够 | 需要大量手动修正 | 提供便捷的手动编辑，持续优化提示词 |
| 画布性能（1000+ 卡片） | 卡顿 | 虚拟滚动 + 按需渲染 |
| Gemini API 在国内访问 | 风格提取功能不可用 | Workers 代理 API 调用，或用国内模型替代 |
